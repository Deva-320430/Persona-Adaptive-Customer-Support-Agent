# Persona-Adaptive Support Agent

## Project Overview

A RAG-powered support chatbot that:
1. Detects the user's persona (Technical Expert, Frustrated User, Business Executive)
2. Retrieves relevant knowledge base content
3. Generates a persona-adapted, context-grounded response
4. Escalates to a human agent when confidence is low, dissatisfaction builds up, or sensitive keywords appear
5. Produces a structured handoff summary for the human agent on escalation

Built with **Google Gemini** (`google-genai` SDK) end-to-end: `gemini-2.5-flash`
for generation and persona classification, `gemini-embedding-001` for
retrieval embeddings — everything runs on a single API key, no separate
local model downloads required.

## Tech Stack

- **Python**: 3.10+ (project targets a standard Python runtime)
- **LLM SDK**: `google-genai` (Gemini API access)
- **Embedding model**: `gemini-embedding-001`
- **Generation model**: `gemini-2.5-flash`
- **Vector DB**: FAISS via `faiss-cpu`
- **UI**: `streamlit`
- **Environment management**: `python-dotenv`
- **Document parsing**: `pypdf`, `python-docx`
- **Numerical processing**: `numpy`

## Persona Detection Strategy

- **Classification method**: a Gemini prompt is used to classify each user message into one of three personas: `Technical Expert`, `Frustrated User`, or `Business Executive`.
- **Prompt design**: the prompt asks for exactly one persona name, with a strict output format to reduce ambiguity.
- **Rules used**: if the model output is invalid, the code falls back to simple keyword matching and then defaults to `Frustrated User` when needed.

## RAG Pipeline Design

- **Chunking strategy**: documents are split into overlapping character windows using `CHUNK_SIZE=500` and `CHUNK_OVERLAP=100`.
- **Embedding model**: queries and document chunks are embedded with `gemini-embedding-001`.
- **Vector database choice**: FAISS stores the chunk vectors and metadata for fast similarity lookup.
- **Retrieval strategy**: the top `TOP_K` matching chunks are returned using cosine similarity on normalized vectors.

## Architecture Diagram

```

    ▼

    │
    ▼
RAG Retriever (FAISS + Gemini embeddings, cosine similarity)
    │
    ▼
Response Generator (Gemini, persona-adapted prompt, grounded in context)
    │
    ▼

    │  - low retrieval confidence
    │  - repeated dissatisfaction signals
    │  - sensitive keyword match (billing, legal, security)
    │
 ┌──┴──────────┐

 ▼             ▼
Answer    Escalate → Handoff Summary (persona, history, sources, reason)
```

## Environment Variables

Create a `.env` file in the project root using the example below:

```env
GEMINI_API_KEY=your_api_key_here
GEMINI_MODEL=gemini-2.5-flash
EMBEDDING_MODEL=gemini-embedding-001
CHUNK_SIZE=500
CHUNK_OVERLAP=100
TOP_K=3
CONFIDENCE_THRESHOLD=0.35
DISSATISFACTION_LIMIT=2
```

Required variable:
- `GEMINI_API_KEY` — your Google Gemini API key

Optional tuning variables:
- `GEMINI_MODEL` — override the generation model
- `EMBEDDING_MODEL` — override the embedding model
- `CHUNK_SIZE` / `CHUNK_OVERLAP` — control chunking behavior
- `TOP_K` — number of retrieved chunks to use
- `CONFIDENCE_THRESHOLD` — escalation cutoff for low retrieval confidence
- `DISSATISFACTION_LIMIT` — number of frustration signals before escalation

## Setup Instructions

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Add your Gemini API key

```bash
cp .env.example .env
# then edit .env and paste your key:
# GEMINI_API_KEY=your_actual_key
```

Get a free key at https://aistudio.google.com/apikey

### 3. Build the knowledge base index

This calls the Gemini embedding API for each chunk in `data/` — uses your
existing API key, no separate model download needed.

```bash
python src/ingestion.py
```

You should see output ending in `Ingestion complete.` and two new files in
`vectorstore/`: `index.faiss` and `metadata.json`.

### 4. Run it

**CLI:**
```bash
python src/chatbot.py
```

**Streamlit UI:**
```bash
streamlit run app.py
```

## Verifying the pipeline without an API key

`tests/test_pipeline_offline.py` mocks both the Gemini generation call and
the retriever (which would otherwise also call the Gemini embedding API),
so you can confirm persona detection → response generation → escalation →
handoff are wired correctly before spending any API quota:

```bash
python tests/test_pipeline_offline.py
```

## How escalation works

Retrieval uses **cosine similarity** (via L2-normalized embeddings + a FAISS
inner-product index), so **higher scores mean better matches** — this is the
opposite direction from raw FAISS L2 distance, which is a common source of
bugs in RAG pipelines if you copy code without checking. Escalation triggers
on any of:

- Top retrieval score below `CONFIDENCE_THRESHOLD` (default `0.35`)
- 2+ dissatisfaction-signal messages in the conversation (default limit)
- A sensitive keyword match (`billing dispute`, `chargeback`, `lawsuit`, `account hacked`, etc.)

All thresholds live in `src/config.py`.

## Known Limitations

- The system depends on the quality and completeness of the knowledge base documents in `data/`.
- Retrieval quality may degrade for questions that require reasoning beyond the provided documents.
- The persona classifier and response generator rely on external Gemini API calls, so latency and quota limits can affect performance.
- The current chunking and retrieval heuristics are simple and may need tuning for larger or more diverse datasets.
- Future improvements could include hybrid search, query rewriting, better citation handling, and more advanced escalation rules.

## Notes on design decisions

- **Gemini embeddings (`gemini-embedding-001`) instead of a local model**:
  everything runs through your existing API key — no separate model
  download, no Hugging Face dependency, works in network-restricted
  environments. Tradeoff: each ingestion run and each query consumes a
  small amount of API quota (free tier has rate limits — see
  `src/llm_client.py`'s `embed_texts` batching if you hit 429s).
- **`google-genai` instead of the legacy `google-generativeai` package**: the
  legacy package is fully deprecated by Google as of 2026; `google-genai` is
  the current supported SDK.
- **Model names are one-line config changes** (`src/config.py` →
  `GEMINI_MODEL` / `EMBEDDING_MODEL`) so you can swap models without
  touching any other code. Confirmed available on this project's API key
  as of June 2026: `gemini-2.5-flash`, `gemini-3.5-flash`,
  `gemini-3.1-flash-lite`, `gemini-embedding-001`.
